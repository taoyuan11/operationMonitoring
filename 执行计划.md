# 远程资源监控系统 MVP 执行计划

## 1. 产品目标

构建一个自托管的服务器/电脑资源监控系统，用于 1-50 台实例的小规模运维场景。

核心闭环：

1. 实例端 Agent 启动并生成本地身份。
2. Agent 向后端注册，进入待审批状态。
3. 管理员批准实例。
4. Agent 每 5 秒上报 CPU、内存、磁盘、网络、GPU 可用信息、在线时长等指标。
5. 游客可公开查看只读看板。
6. 管理员可编辑实例、停用/删除实例、配置快捷操作、打开 Web 终端执行命令。
7. 管理操作、命令执行、SSH/Web 终端会话写入日志。

## 2. 技术边界

- 前端：Vue 3 + Vite，单页监控控制台。
- 后端：Rust + Axum + SQLx + SQLite。
- 实例端：Rust Agent，支持 Linux、Windows、macOS。
- 数据库：首版使用单机 SQLite，默认指标保留 30 天。
- 通信：
  - HTTP API 用于登录、审批、实例管理、指标查询、设置和命令配置。
  - WebSocket 用于 Agent 指标、心跳、快捷命令和交互式 PTY 终端的持久复用通道。

## 3. 权限模型

- 游客：
  - 无需登录。
  - 可访问公开看板和历史指标。
  - 不可查看密钥、命令日志、SSH 信息、管理设置。
- 管理员：
  - 通过环境变量或启动参数设置账号密码。
  - 登录后获得服务端 session cookie。
  - 可审批实例、编辑实例、停用/删除实例、配置快捷命令、打开 Web 终端、查看日志和修改保留天数。

## 4. 首版数据模型

后端 SQLite 至少包含：

- `instances`：已批准实例、备注、地区、系统信息、最后上报时间、停用状态。
- `pending_instances`：待审批实例。
- `metrics`：实例指标样本。
- `settings`：系统设置，例如 `retention_days`。
- `commands`：管理员配置的白名单快捷操作。
- `command_jobs`：命令执行任务和结果。
- `ssh_sessions`：Web 终端会话记录。
- `action_logs`：审批、编辑、删除、命令等操作日志。

## 5. API 契约

公共接口：

- `GET /api/public/instances`
- `GET /api/public/instances/{id}/metrics`

管理接口：

- `POST /api/admin/login`
- `POST /api/admin/logout`
- `GET /api/admin/me`
- `GET /api/admin/pending-instances`
- `POST /api/admin/pending-instances/{id}/approve`
- `POST /api/admin/pending-instances/{id}/reject`
- `PATCH /api/admin/instances/{id}`
- `POST /api/admin/instances/{id}/disable`
- `DELETE /api/admin/instances/{id}`
- `GET /api/admin/settings`
- `PUT /api/admin/settings`
- `GET /api/admin/commands`
- `POST /api/admin/commands`
- `DELETE /api/admin/commands/{id}`
- `POST /api/admin/instances/{id}/commands/{command_id}/run`
- `GET /api/admin/instances/{id}/terminal/ws`
- `GET /api/admin/jobs`
- `GET /api/admin/logs`

实例接口：

- `POST /api/agent/register`
- `POST /api/agent/report`（保留兼容；新版 Agent 通过 WebSocket 上报）
- `GET /api/agent/ws`

## 6. 前端页面

首页即产品主界面：

- 顶部状态栏：系统名、刷新、管理员状态。
- 概览指标：实例总数、在线实例、平均 CPU、平均内存。
- 实例看板：
  - 默认网格视图。
  - 支持行视图切换。
  - 展示实例名称、地区、备注、系统、在线状态、在线时长、CPU、GPU、内存、硬盘、网络和最后上报时间。
- 管理侧栏：
  - 未登录时显示管理员登录。
  - 登录后显示审批、命令、设置、日志四个管理标签。
- 弹窗：
  - 实例编辑弹窗。
  - Web 终端弹窗。

## 7. 实例端行为

- 默认服务器地址：`http://127.0.0.1:8080`。
- 可通过 `--server` 或 `OM_SERVER` 指定后端地址。
- 本地身份默认写入系统配置目录，也可通过 `OM_AGENT_ID_FILE` 指定。
- 上报间隔默认 5 秒，可通过 `OM_REPORT_INTERVAL` 修改。
- GPU 采集首版允许返回未知，不阻塞其他指标。
- 收到后端命令后通过系统 shell 执行，并回传 stdout、stderr、退出码。

## 8. 验收标准

后端：

- `cargo check` 通过。
- 能初始化 SQLite 表。
- 未审批实例只进入待审批列表，不写入指标。
- 审批后实例指标写入并可公开查询。
- 管理接口必须登录。
- 命令下发、执行结果和操作日志可查询。

实例端：

- `cargo check` 通过。
- 首次启动生成稳定身份。
- 后端不可达时持续重试。
- 待审批时继续发送注册请求。
- 审批后建立 WebSocket，并通过该连接上报指标、接收命令和承载 PTY 会话。
- WebSocket 断开或心跳超时后，后端立即将实例标记为离线。

前端：

- `pnpm build` 通过。
- 游客只能看到看板。
- 管理员登录后出现管理按钮和侧栏。
- 可审批实例、编辑实例、配置快捷操作、打开 Web 终端。
- 移动端和桌面端无明显内容重叠。

## 9. 后续增强

- 增加告警规则、通知渠道和阈值配置。
- 增加历史趋势图、聚合降采样和更长周期存储。
- 增加 Agent 自动升级和服务安装脚本。
